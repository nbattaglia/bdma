S3. Declaración de reglas de interacción (eventos compartidos, ACL, contratos esperados). La comunicación debe ser explicita, controlada (sin acoplamientos indeseados) y mantenible
•	Eventos compartidos clave:
o	EnvioRegistrado (BC03) → BC04, BC05
o	RevisionRegistrada (BC04) → BC05
o	DecisionEmitida (BC05) → BC03
o	ReglasPublicadas (BC06) → BC03, BC04, BC05
o	PerfilSincronizado (BC08) → BC02
o	PublicacionesImportadas (BC08) → BC02, BC03
•	Contratos API esperados:
o	BC01 expone Auth API (login/registro)
o	BC06 expone API de reglas y roles
o	BC02 expone API de perfil resumido (solo lectura para otros contextos)
•	Comunicación explícita:
o	Mensajería asíncrona para eventos de dominio inter-contexto
o	ACL en integraciones externas para evitar acoplamiento semántico
